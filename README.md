# Grocery Store Queuing Simulation

An agent-based simulation of customer queuing behaviour at grocery store checkout counters, built with **NetLogo 7.0.2**. The model captures dynamic lane management, customer jockeying, peak-time demand surges, shift-based staffing changes, and multiple checkout lane types — making it suitable for studying real-world retail queue dynamics.

> **Live Demo:** [Run in browser](https://c-yasitha.github.io/grocery-store-simulation/)

## Overview

This simulation models a grocery store with multiple checkout lanes operating over one or more full operating days. It investigates how queue length, customer wait time, cashier utilisation, and throughput are affected by:

- **Lane type mix** — cash-only, card-only, express, and self-checkout
- **Customer jockeying** — customers switching to shorter queues
- **Dynamic lane conversion** — idle regular lanes temporarily become express lanes under high express demand
- **Shift-based staffing** — up to 4 shifts per day with configurable lane counts and arrival rates per shift
- **Peak periods** — configurable morning/evening rush multipliers applied to the arrival rate

---

## Features

| Feature | Description |
|---------|-------------|
| 4 lane types | Cash-only, Card-only, Express (<5 items), Self-checkout (card only) |
| Customer jockeying | Agents switch lanes when a neighbour's queue is shorter by >2 positions |
| Dynamic express conversion | Regular lanes auto-convert to express when express queue > 15 and a regular lane is idle |
| Shift system | Up to 4 shifts per operating day; each shift sets its own arrival rate, service time, and cashier counts |
| Peak periods | Two configurable peak windows with independent arrival rate multipliers |
| Stochastic arrivals | Poisson-like arrivals scaled to customers-per-hour rate |
| Payment-method timing | Separate configurable distributions for cash and card payment durations |
| Statistics | Average/P95 wait time, throughput (customers/min), cashier utilisation, average queue length |
| Web interface | Browser-based sidebar for configuring all parameters and running the model |

## Getting Started

### Option A — Run in Browser (No Installation)

Visit the live demo: **[https://c-yasitha.github.io/grocery-store-simulation/](https://c-yasitha.github.io/grocery-store-simulation/)**

The sidebar on the right lets you configure all parameters. Click **"Copy Command"**, paste it into the NetLogo Web command centre panel, press **Enter**, then click **Go**.

### Option B — Run Locally via HTTP Server

The web interface must be served over HTTP (not opened as a local file) for all features to work correctly.

```bash
git clone git@github.com:C-Yasitha/grocery-store-simulation.git
cd grocery-store-simulation/web
python3 -m http.server 8080
```

Open **http://localhost:8080** in your browser.

### Option C — Run in NetLogo Desktop

1. Install [NetLogo 7.0.2](https://ccl.northwestern.edu/netlogo/download.shtml)
2. Open `grocery_v1.3.nlogox`
3. Set parameters via the command centre using the command generated by the web sidebar, or set globals manually
4. Click **Setup** then **Go**

## Agent Types

### Customers
Arrive stochastically, choose a checkout lane (or jockey to a shorter one), wait in queue, get served, and leave.

| Attribute | Range | Description |
|-----------|-------|-------------|
| `items-in-hand` | 1–15 | Number of grocery items |
| `payment-method` | `"cash"` / `"card"` | Determines eligible lanes and payment duration |
| `my-cashier` | agent ref | Assigned checkout cashier |
| `queue-position` | integer | Position in queue (1 = next to be served) |

### Cashiers
Stationary agents at checkout counters. Process customers in FIFO order. Can be hidden (closed) during shifts where they are not staffed.

| Attribute | Values | Description |
|-----------|--------|-------------|
| `cashier-type` | `"cash"` / `"card"` / `"express"` / `"self-checkout"` | Lane type |
| `busy?` | boolean | Whether actively serving a customer |
| `remaining-service-time` | ticks | Time left to finish current customer |
| `express?` | boolean | `true` if temporarily converted to express |

## Lane Types

| Lane | Colour | Accepts | Notes |
|------|--------|---------|-------|
| Cash-only | Green | Cash customers | Standard service time |
| Card-only | Orange | Card customers | Standard service time |
| Express | Blue | Any customer with < 5 items | Fixed 60-second service |
| Self-checkout | Magenta | Card customers, any items | Configurable scan + payment time |

## Configuration Parameters

All parameters are set via the web sidebar (which generates a single command) or directly in the NetLogo command centre.

### Timing & Scale

| Parameter | Default | Description |
|-----------|---------|-------------|
| `num-rounds` | 1 | Number of full operating days to simulate |
| `operating-hours` | 16 | Hours per operating day |
| `num-shifts` | 1 | Shifts per day (1–4) |

### Lanes (Base / Shift 1)

| Parameter | Default | Description |
|-----------|---------|-------------|
| `num-cash-counters` | 3 | Cash-only lanes open in shift 1 |
| `num-card-counters` | 3 | Card-only lanes open in shift 1 |
| `num-express-counters` | 1 | Express lanes open in shift 1 |
| `num-self-checkout-counters` | 2 | Self-checkout kiosks open in shift 1 |

### Arrivals & Service

| Parameter | Default | Description |
|-----------|---------|-------------|
| `arrival-rate` | 60 | Customers per hour (shift 1) |
| `avg-service-time` | 300 | Average total service time in seconds (shift 1) |
| `prob-cash` | 0.5 | Probability a customer pays cash |
| `service-variability` | 0.2 | Coefficient of variation for service time |
| `cash-payment-mean` | 11 | Mean cash payment duration (seconds) |
| `cash-payment-sd` | 3 | SD of cash payment duration |
| `card-payment-mean` | 6 | Mean card payment duration (seconds) |
| `card-payment-sd` | 2 | SD of card payment duration |

### Self-Checkout

| Parameter | Default | Description |
|-----------|---------|-------------|
| `self-checkout-scan-time` | 5 | Seconds per item scanned |
| `self-checkout-payment-time` | 8 | Card payment time at self-checkout |

### Peak Periods

| Parameter | Default | Description |
|-----------|---------|-------------|
| `peak1-start-min` / `peak1-end-min` | 0 | Start/end of first peak (minutes from open; 0 = disabled) |
| `peak1-multiplier` | 0 | Arrival rate multiplier during peak 1 |
| `peak2-start-min` / `peak2-end-min` | 0 | Start/end of second peak |
| `peak2-multiplier` | 0 | Arrival rate multiplier during peak 2 |

### Per-Shift Lists (when `num-shifts` > 1)

| Parameter | Description |
|-----------|-------------|
| `shift-num-cash` | List of cash counter counts per shift |
| `shift-num-card` | List of card counter counts per shift |
| `shift-num-express` | List of express counter counts per shift |
| `shift-num-self-checkout` | List of self-checkout counts per shift |
| `shift-arrival-rates` | Arrival rate (customers/hr) per shift |
| `shift-avg-service-times` | Average service time (seconds) per shift |

## Output Statistics

| Metric | Description |
|--------|-------------|
| `avg-wait-time` | Mean customer wait time in queue (seconds) |
| `p95-wait-time` | 95th-percentile wait time (interpolated) |
| `throughput` | Customers served per minute |
| `utilization` | Average fraction of open cashiers actively serving |
| `avg-queue-length` | Mean number of customers queued across all lanes |
